I0 Checklist - Technology

Technology

Architecture & Design
  1. Has the project architecture and design been reviewed by an ADC Architect or Consultant level developer?
    1. If no, please explain. Answers of "N/A" will not be accepted
  2. Will there be technical debt as a result of this project?
  3. Is there agreement with the BSA on ownership and management of Technical Debt items?
Software Engineering Practices
  1. Is the line using code quality standards? Have you considered how they will be enforced?
  2. Do you have a source code control strategy defined? This should include comment format, branching & merging.
  3. Are you planning to use automated build?
    1. Do you have automated build scripts that have been tested?
  4. Are you planning to use automated deploy?
    1. Do you have automated deploy scripts that have been tested?
  5. Do you have a continuous integration strategy defined? This should include how often you will integrate and how you will manage failed builds.
Environments
  1. Do you have all the necessary development tools installed, configured and ready for iteration 1?
  2. Have all environments been configured and are ready for use?
    1. Development
    2. Test
    3. Production
  3. Do you have access to logs in each environment?
  4. Have you engaged code migration to establish/update/validate migration scripts?
Test
  1. Do you have a documented plan or appoach on what will be automated?
  2. Do you have all the necessary test automation development tools installed, configured and ready for iteration 1?
  • Will you run the test automation on a continuous integration server?
  • Are you using existing test automation pattern(s)?
    1. If no, has your automation design been reviewed and approved by the Architect or H-Band Developer? Please explain.
